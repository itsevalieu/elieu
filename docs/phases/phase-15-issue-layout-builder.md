# Phase 15 — Issue Layout Builder (Hybrid Curation + Algorithm)

## Overview

Add a hybrid issue layout system where the admin curates featured, sidebar,
and positioned posts via a drag-and-drop builder with live preview, while the
algorithm fills remaining grid slots from subcategories automatically.

## Current State

- Posts have `issueId` (Long) and `layoutHint` (String) on the Post entity
- Issues have metadata (title, month, year, status, layoutPreference) but no post-ordering mechanism
- The front page is rendered by `buildSections()` in `NewspaperGrid.tsx` purely algorithmically (picks highest-viewCount post per subcategory)
- Admin can assign posts to issues via a dropdown in `AdminPostForm.tsx`, but there is no visual arrangement
- `@hello-pangea/dnd` is already in `frontend/admin/package.json` but unused

## Tasks

### 1. Database: `issue_posts` Join Table

New Flyway migration creating a dedicated join table instead of relying on
`Post.issueId` alone. This keeps position data issue-specific (a post could
appear in multiple issues with different positions).

```sql
CREATE TABLE issue_posts (
  id            BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  issue_id      BIGINT NOT NULL REFERENCES issues(id) ON DELETE CASCADE,
  post_id       BIGINT NOT NULL REFERENCES posts(id) ON DELETE CASCADE,
  position      VARCHAR(20) NOT NULL DEFAULT 'default',
  sort_order    INT NOT NULL DEFAULT 0,
  created_at    TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  UNIQUE(issue_id, post_id)
);
```

**`position` values**: `featured`, `wide`, `full`, `default`, `compact`, `sidebar`, `quote`
— maps directly to the existing `CardVariant` type in the frontend.

### 2. Backend: Entity, Repository, DTOs

- **`IssuePost`** entity with `id`, `issueId`, `postId`, `position`, `sortOrder`, `createdAt`; `@ManyToOne` to Issue and Post.
- **`IssuePostRepository`**: `findByIssueIdOrderBySortOrder(Long)`, `deleteByIssueId(Long)`.
- **`IssueLayoutItem`** request DTO: `{ postId, position, sortOrder }`.
- **`IssueLayoutResponse`** response DTO: full post data + position metadata.

### 3. Backend: API Endpoints + Hybrid Render

**New endpoints on `IssueController`:**
- `GET /api/admin/issues/{id}/layout` — returns curated posts with positions
- `PUT /api/admin/issues/{id}/layout` — bulk-save layout (array of `IssueLayoutItem`)

**`IssueService` additions:**
- `getLayout(Long issueId)` — returns curated `IssuePost` list
- `saveLayout(Long issueId, List<IssueLayoutItem> items)` — replaces all `issue_posts` rows atomically

**Hybrid render logic** (new `IssueLayoutService` or in `PostService`):
- When rendering an issue: load curated posts first, then fill remaining subcategory slots algorithmically (skip already-curated subcategories)
- Expose via `GET /api/issues/{slug}` response, adding a `layout` array with position metadata

### 4. Admin UI: Issue Layout Builder

**New page**: `frontend/admin/src/app/(admin)/issues/[id]/layout/page.tsx`

**Layout:**
- **Left panel**: Post picker — searchable/filterable list of published posts (by category, subcategory, title)
- **Right panel**: Layout board with drop zones for each position type

**Drop zones** (using `@hello-pangea/dnd`):
- **Featured** (1 slot) — the hero article
- **Wide** (2 slots) — 2-column span cards
- **Main Grid** (unlimited) — default-sized cards, reorderable
- **Sidebar** (unlimited) — sidebar cards, reorderable
- **Compact** (unlimited) — brief/compact cards

**Each card in a zone shows:**
- Post title, subcategory, thumbnail (if available)
- Drag handle for reordering
- Remove button (X) to unassign

**Auto-fill button:**
- Runs the same algorithm as the front page to fill empty zones
- Admin can then rearrange or swap posts

**Preview button:**
- Opens a modal or side panel showing a simplified newspaper layout preview using the same `ExcerptCard` variants

**Save:**
- `PUT /api/admin/issues/{id}/layout` with the full layout array

### 5. Frontend Rendering Changes

**`NewspaperGrid.tsx`:**
- Accept an optional `layout` prop (curated `IssuePost[]` with positions)
- If `layout` is provided, use it for post placement instead of `buildSections()`
- Fill any remaining slots algorithmically (hybrid mode)
- If no layout is provided (e.g., front page with no specific issue), keep current behavior

**Issue page** (`/issues/[slug]`):
- Fetch issue with layout data
- Pass layout to `NewspaperGrid`
